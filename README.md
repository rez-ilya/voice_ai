Для развертывания и полноценного функционирования гибридного голосового ассистента в системе необходимы внешние зависимости, которые обеспечивают нечеткую логику, системные запросы и интеграцию с искусственным интеллектом Сбера. К ним относятся официальный SDK gigachat для аппаратного вызова инструментов через Function Calling, библиотека rapidfuzz для мгновенного локального сравнения строк по алгоритмам Левенштейна, модуль requests для получения погодных данных с открытых API и официальный клиент wikipedia для быстрого извлечения энциклопедических справок. Все остальные используемые модули, такие как os, ctypes, imaplib или webbrowser, входят в стандартный дистрибутив языка Python и не требуют отдельной загрузки.

Установить все пакеты можно поштучно с помощью стандартного менеджера пакетов посредством команды pip install gigachat rapidfuzz requests wikipedia. Перед запуском главного скрипта необходимо открыть конфигурационный файл config.py и указать там валидные авторизационные данные от GigaChat API, а также логин и пароль от почтового ящика для работы IMAP-клиента. После этого следует подключить микрофон, проверить настройки звуковых устройств в Windows и инициализировать программу через командную строку, выполнив команду python main.py. Скрипт подаст звуковой сигнал готовности и перейдет в режим бесконечного прослушивания эфира.

В процессе работы ассистент демонстрирует гибкое гибридное поведение в зависимости от семантики запроса. Если пользователь произносит простые фразы вроде «Сколько сейчас времени» или «Который час», локальный классификатор мгновенно перехватывает управление и озвучивает системное время без обращения к серверам ИИ. Точно так же локально обрабатываются запросы «Погода в Томске», при которых скрипт делает прямой HTTP-запрос, и команда «Проверь почту», запускающая SSL-соединение с почтовым сервером для чтения заголовков. Приказы запустить локальный софт, такие как «Открой блокнот», «Открой калькулятор» или «Открой браузер», обрабатываются через встроенный словарь жестких соответствий, а простые справочные вопросы вроде «Что такое черная дыра» автоматически очищаются от мусорных слов и направляются в локальный модуль Википедии.

Если же пользователь ставит перед системой сложную задачу, локальные сценарии возвращают пустой результат и управление делегируется нейросети. Например, при запросе «Открой галерею» GigaChat понимает контекст Windows и через механизм Function Calling возвращает точную системную схему ms-photos, открывая встроенное приложение. При поисковых командах вроде «Найди в гугле информацию о черной дыре» или «Поищи в гугле рецепт шарлотки» искусственный интеллект выделяет суть запроса, формирует веб-ссылку и открывает окно браузера на нужной странице. Аналогичным образом обрабатываются мультимедийные запросы «Включи видео о черной дыре» или «Включи видео с котиками», где ИИ автоматически активирует инструмент с аргументом youtube для прямой выдачи видеороликов. Если пользователь задает отвлеченный вопрос наподобие «Зачем человеку спать» или произносит фразы вежливости вроде «Спасибо за помощь», ассистент отключает системные функции и переходит в режим интеллектуальной болталки, выдавая емкие ответы в два предложения. Работа конвейера завершается при распознавании прощальной команды «Пока», после чего Джарвис вежливо прощается и корректно останавливает бесконечный цикл.
